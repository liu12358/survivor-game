# 第一部分：文档基础框架

## 1. 游戏愿景与差异化定位

### 1.1 一句话产品愿景

> 在星屑坠落的荒原，操控法师在无尽魔物潮中杀出一条血路——移动即战术，Build即宿命，活到15分钟即是传奇。

### 1.2 目标玩家画像

| 维度 | 描述 |
|------|------|
| 年龄 | 12-35岁 |
| 游戏经验 | 轻度到中度玩家，喜欢短时快节奏 |
| 核心动机 | 爽感释放、Build探索、分数攀比 |
| 单局时长 | 5-20分钟（主力15分钟） |
| 平台偏好 | PC为主，移动端碎片化游玩 |
| 年龄分级 | ESRB E10+ / PEGI 7（卡通像素暴力，无血腥） |

### 1.3 竞品差异化对比

| 维度 | Vampire Survivors | Brotato | 星屑割草（我们） |
|------|-------------------|---------|-------------------|
| 视角 | 俯视像素 | 俯视像素 | **2.5D俯视+粒子特效** |
| 攻击方式 | 全自动 | 手动瞄准+自动 | **全自动，但武器进化改变形态** |
| 成长系统 | 武器+被动→超武 | 武器+道具叠加 | **武器进化树+超武合成+词条随机** |
| 角色差异化 | 初始武器不同 | 角色特性明显 | **角色专属被动+解锁剧情** |
| 地图 | 多张固定地图 | 小竞技场 | **Roguelike随机地图种子+地形交互** |
| 局外成长 | 金币永久强化 | 角色解锁 | **完整Meta Progression+图鉴收集** |
| 美术风格 | 极简像素 | 卡通风格 | **星尘魔法主题+发光粒子统一风格** |
| BOSS设计 | 固定时间刷 | 波次BOSS | **多阶段BOSS+专属弹幕+剧情台词** |

### 1.4 核心差异化卖点

1. **武器进化树** — 6种武器各有独立升级分支，不是简单数值+1
2. **超武合成** — 特定武器+特定被动满级→终极超武（核心爽点）
3. **词条随机** — 升级时有概率出现稀有/史诗词条（额外移速、吸血、暴击）
4. **星尘魔法主题** — 统一的星屑粒子+光效美术语言
5. **多阶段BOSS** — 暗影领主有3个阶段，不是单纯血牛
6. **完整局外成长** — Meta Progression 让每局都有意义

```

敌色调（魔物/危险）：  #FF5252 (猩红) #FF6E40 (暗橙) #4A148C (暗紫)

特效色（粒子/发光）：  #E1F5FE (淡蓝光) #F3E5F5 (淡紫光) #FFF9C4 (淡金光)
```

### 2.3 画风统一规则

- **线条**：1px 黑色描边（主要角色），0px（子弹/粒子）
- **发光**：统一使用 Godot WorldEnvironment glow 后处理，所有发光物体用 `modulate` 调色
- **粒子**：统一通过 GPU Particle2D，材质使用 `additive` 混合模式
- **阴影**：统一在角色脚底做 8×4 px 椭圆暗影（不随旋转）

## 4. 完整世界观设定

### 4.1 世界背景

> **星屑纪元 第7纪元**  
> 天空的星辰正在一颗颗坠落。每一颗坠星都化为"星屑"——既是魔力的源泉，也是打开暗影裂隙的钥匙。  
> 暗影领主从裂隙中苏醒，率领魔物潮席卷大地。  
> 最后的星尘法师集结于「守望荒原」，以星屑之力对抗暗影侵蚀。

### 4.2 星屑设定

- **星屑**：坠星碎片，蕴含原始魔力，是法师的能源，也是魔物的诱饵
- **经验宝石 = 星屑碎片**：击杀魔物后释放的微量星屑
- **BOSS**：吸收了巨量星屑而变异的远古魔物
- **15分钟存活 = 封印裂隙**：撑到星屑能量重新稳定，裂隙关闭

### 4.3 势力设定

| 势力 | 描述 |
|------|------|
| 星尘法师 | 操控星屑魔力的守护者，玩家阵营 |
| 暗影魔物 | 从裂隙涌出的扭曲生物，被星屑吸引 |
| 暗影领主 | 裂隙之主，远古被封印的魔君，星屑坠落解开了封印 |
| 守望者（未解锁） | 隐藏角色阵营，远古中立势力 |

### 4.4 场景环境叙事

| 地图 | 环境叙事线索 |
|------|-------------|
| 守望荒原（初始） | 散落的破碎石碑、废弃法师塔残骸、星屑陨坑 |
| 暗影裂谷（Phase 3） | 被暗影侵蚀的扭曲树木、流淌紫色岩浆的裂缝、巨大骨架 |
| 星落祭坛（Phase 3+） | 古代祭坛遗迹、悬浮的星屑结晶、破碎的封印阵 |

## 8. 玩家完整状态系统

### 8.1 玩家状态机

```
                    ┌──────────┐
        游戏开始 → │  正常态   │ ← 升级结束/受伤无敌结束
                    └─────┬────┘
                          │
        ┌─────┬──────┬────┼──────┬──────┬─────┐
        ↓     ↓      ↓    ↓      ↓      ↓     ↓
     受伤无敌 升级暂停  死亡  减速  灼烧  冰冻  眩晕
        │     │      │    │      │      │     │
        └─────┘      │    └──────┴──────┴─────┘
          │          │          │
     0.5秒后恢复   游戏结束   持续N秒后恢复
```

### 8.2 异常状态详细规则

| 状态 | 触发来源 | 效果 | 持续时间 | 叠加规则 |
|------|----------|------|----------|----------|
| 受伤无敌 | 玩家被敌人碰到 | 无视碰撞伤害 | 0.5s | 不可叠加，刷新计时 |
| 升级暂停 | 升级触发 | 游戏时间暂停 | 直到选择 | 完全暂停（非慢动作） |
| 死亡 | HP ≤ 0 | 游戏结束 | 永久 | — |
| 减速 | 冰晶风暴/冰系敌人 | 移速-30% | 2.0s | 可叠加至-60%上限 |
| 灼烧 | 烈焰环/火系敌人 | 每秒扣2%最大HP | 3.0s | 可叠加至3层(6%/s) |
| 冰冻 | 冰晶风暴满级 | 完全冻结 | 1.5s | 不可叠加 |
| 眩晕 | BOSS重击 | 无法移动 | 1.0s | 不可叠加 |
| 中毒 | 毒雾/毒系敌人 | 每秒扣1.5%最大HP | 4.0s | 可叠加至5层(7.5%/s) |

### 8.3 回血机制

| 回血来源 | 效果 | 触发条件 | 上限 |
|----------|------|----------|------|
| 回血球（掉落物） | 回复 20% 最大HP | 拾取 | 不超过max_hp |
| 吸血词条 | 造成伤害的 5% 转为HP | 所有武器伤害 | 不超过max_hp |
| 每秒回血被动 | +2 HP/s（每级+2） | 持续生效 | 不超过max_hp |
| 击杀回血 | 击杀回复 3 HP | 击杀触发 | 不超过max_hp |
| 受伤回血 | 受伤后3秒内每秒+5 HP | 受伤触发 | 不超过max_hp |

### 8.4 护盾机制

| 参数 | 值 |
|------|-----|
| 护盾类型 | 临时额外血条，优先于HP消耗 |
| 获取方式 | 护盾球掉落、特定被动技能 |
| 护盾上限 | 50% 最大HP |
| 衰减规则 | 不随时间衰减，受击时先扣护盾再扣HP |
| 护盾与无敌帧 | 护盾被击破不触发无敌帧（只扣HP时触发） |

### 8.5 暴击体系

| 参数 | 值 |
|------|-----|
| 基础暴击率 | 0%（初始） |
| 暴击伤害倍率 | 1.5× |
| 暴击来源 | 被动「锐眼」每级+5%暴击率；武器特定升级；词条 |
| 暴击率上限 | 60% |
| 暴击伤害上限 | 3.0× |
| 暴击视觉 | 黄色伤害数字 + "!" + 略微放大 |

### 8.6 伤害类型

| 类型 | 颜色 | 受什么影响 | 特殊 |

## 11. 伤害计算统一公式

### 11.1 通用伤害公式

```
┌─────────────────────────────────────────────────────┐
│ 最终伤害 = (基础伤害 × 伤害倍率) × 暴击判定           │
│           - 敌方护甲减免 × 伤害穿透率                 │
│           × 伤害类型修正                              │
│                                                     │
│ 其中：                                               │
│   伤害倍率 = 1.0 + 所有增伤buff之和                   │
│   暴击判定 = 暴击率roll → 成功则 × 暴击倍率           │
│   敌方护甲减免 = max(0, 敌方护甲 - 玩家护甲穿透)       │
│   伤害类型修正：物理=1.0, 魔法=1.0(无视护甲),         │
│                 DOT=直接取值, 真实=1.0(无视一切)       │
└─────────────────────────────────────────────────────┘
```

### 11.2 具体伤害计算伪代码

```gdscript
func calculate_damage(base_dmg: float, damage_type: int, 
                       attacker_crit_chance: float, attacker_crit_mult: float,
                       attacker_dmg_bonus: float, attacker_penetration: float,
                       enemy_armor: float) -> Dictionary:
    var result = {"damage": 0.0, "is_crit": false, "type": damage_type}
    
    # 1. 基础伤害 × 增伤系数
    var dmg = base_dmg * (1.0 + attacker_dmg_bonus)
    
    # 2. 暴击判定
    if damage_type != DAMAGE_DOT:  # DOT不可暴击
        if randf() < attacker_crit_chance:
            dmg *= attacker_crit_mult
            result["is_crit"] = true
    
    # 3. 护甲减免（物理伤害适用）
    if damage_type == DAMAGE_PHYSICAL:
        var effective_armor = max(0, enemy_armor - attacker_penetration)
        dmg = max(1, dmg - effective_armor)  # 最少扣1伤害
    
    # 4. 魔法/真实伤害无视护甲
    result["damage"] = dmg
    return result
```

### 11.3 DOT伤害计算

```gdscript
# DOT不参与暴击，不参与护甲减免
func calculate_dot_damage(dot_base: float, stacks: int, duration: float, interval: float) -> float:
    var ticks = duration / interval
    return dot_base * stacks * ticks
```

### 11.4 近战AOE伤害（烈焰环/环绕剑刃）

```
AOE伤害 = 武器伤害 × (1.0 - 距离/最大范围)  # 距离越远伤害越低
最小伤害 = 武器伤害 × 0.5  # 范围边缘最低50%

### 18.5 数据校验与防篡改

```
保存逻辑：
1. 序列化数据 → JSON字符串
2. 计算 JSON 字符串的 SHA256 哈希
3. 存入 {"data": ..., "checksum": "sha256_hash"}

加载逻辑：
1. 读取文件 → 解析JSON
2. 重新计算data部分的SHA256
3. 比对checksum → 不匹配则提示损坏
4. 匹配 → 正常加载

注：这是基础校验，不防高级修改。单机游戏不过度设防。
```

```

**池容量规划**：

| 粒子特效 | 30 | 100 | 回收最旧特效 |

### 19.2 渲染性能优化

| 优化项 | 方法 |
|--------|------|
| 同屏怪物>100 | 远距离怪物用低帧率动画（4fps→2fps） |

## 30. 暂停逻辑细节

| 场景 | 行为 |
|------|------|
| 升级选卡 | **完全暂停**（`Engine.time_scale = 0`），不等慢动作 |
| ESC暂停 | **完全暂停**，所有物理/计时器/动画停止 |
| BOSS登场动画 | **完全暂停**1.5s（演出时间） |
| 窗口失焦 | **自动完全暂停** |
| 移动端切后台 | **自动完全暂停** |

> 不使用慢动作暂停方式。完全暂停更清晰，玩家不会因为"时间还在走"而焦虑。

## 31. 经验宝石堆叠与吸附

| 规则 | 值 |
|------|------|
| 同屏宝石上限 | 200个 |
| 宝石堆叠 | 同一位置≤5个，超出自动分散 |
| 拾取优先级 | 距离玩家最近的先拾取 |
| 磁铁吸附速度 | 宝石飞向玩家速度 = 400 + 磁铁范围×2 px/s |
| 磁铁范围叠加 | 基础60px + 被动「磁铁」+ Meta强化 + Buff |

## 32. 击杀计数与分数计算

```
单局分数公式：
Score = 击杀数 × 1 
      + 存活秒数 × 2
      + BOSS击杀 × 500
      + 拾取金币 × 10
      + MAX武器数 × 100
      + 难度系数

难度系数：简单=0.7 / 普通=1.0 / 困难=1.5 / 噩梦=2.5 / 星陨=5.0
```

## 33. 游戏结束分支

```
HP归零：
├── 持有复活羽毛 → 弹出复活确认 → 使用 → 继续
├── 无复活 → 弹出结算
└── 移动端激励广告复活 → 弹广告 → 复活（可选，Phase 3后期）

BOSS击杀：
├── 弹出BOSS击杀提示
├── 掉落专属奖励
├── 进入无尽模式（可继续玩到死亡）
└── 手动退出 → 结算（记录BOSS通关）

玩家主动退出（ESC → 退出 → 确认弹窗）：
├── [放弃本局] → 弹出惩罚说明 → 确认 → 触发退出结算
├── [返回游戏] → 关闭弹窗，继续游戏
└── 退出结算：见下方「33.1 主动退出结算规则」

Alt+F4 / 杀进程 / 强制关闭：
└── 无结算弹窗 → 下次启动游戏自动结算（后台记录强制退出时间点）

15分钟存活：
├── 触发通关动画
├── 解锁剑圣（首次）+ 暗影裂谷地图
├── 弹出特殊结算
└── 可选择继续无尽模式
```

### 33.1 主动退出结算规则

**核心原则**：鼓励玩到底，但允许中途退出。退出有惩罚但不完全白玩。

| 退出时间点 | 经验结算 | 金币结算 | 分数结算 | 解锁进度 |
|-----------|----------|----------|----------|----------|
| < 1分钟 | 0% | 0% | 0 | ❌ 不计 |
| 1-3分钟 | 50% | 30% | 30% | ✅ 保留 |
| 3-5分钟 | 70% | 50% | 50% | ✅ 保留 |
| 5-10分钟 | 85% | 70% | 70% | ✅ 保留 |
| 10分钟+ | 100% | 85% | 85% | ✅ 保留 |
| BOSS击杀后 | 100% | 100% | 100% | ✅ 保留 |

**主动退出弹窗文案**：
```
┌──────────────────────────────────────┐
│      确定要退出本局吗？                │
│                                      │
│  已存活：4分23秒                       │
│  击杀数：128 只                       │
│                                      │
│  ⚠ 提前退出将损失：                    │
│    经验值扣除 30%                      │
│    金币扣除 50%                        │
│    本局分数扣除 50%                    │
│                                      │
│  ✅ 以下不受影响：                     │
│    武器/被动解锁进度                    │
│    成就进度                            │
│    击杀计数                            │
│                                      │
│  [放弃本局]  [返回游戏]               │
└──────────────────────────────────────┘
```

**强制退出恢复结算**：
| 场景 | 处理 |
|------|------|
| 系统崩溃/断电 | 下次启动检测到未完成局 → 弹出「检测到上次对局异常退出」→ 自动按退出时间计算结算 |
| 进程被 kill | 同上 |
| 移动端切后台被杀 | 同上（Godot 的 `NOTIFICATION_WM_ABOUT` 触发快速存档） |

**退出结算与正常结算的区别**：
| 维度 | 正常结算（死亡/通关） | 主动退出结算 |
|------|---------------------|-------------|
| 金币倍率 | 100% | 30%-100%（按时长） |
| 分数倍率 | 100% | 30%-100%（按时长） |
| 经验倍率 | 100% | 50%-100%（按时长） |
| 解锁触发 | ✅ | ✅（不受影响） |
| 成就触发 | ✅ | ✅（不受影响） |
| 击杀计数 | ✅ | ✅（完整保留） |
| 弹窗UI | 正常结算面板 | 惩罚说明+结算面板 |
| 记录标签 | 正常 | 标记为「主动退出」 |

**设计意图**：
- 12分钟时退出只损失15%金币，让玩家不觉得"亏了想走"
- 1分钟内退出全无收益，防止刷子行为
- 击杀/成就/解锁永久保留，不因退出而丢失进度
- 惩罚随时间递减，接近通关时退出惩罚极低（鼓励玩到BOSS）
