## 0.2 技术架构现状

```
Autoload 单例（全局，project.godot 注册）
├── EventBus     纯信号事件总线 —— 战斗/UI/存档全解耦
├── GameState    局内运行时数据 + 局外持久镜像 + 难度配置表
├── AudioManager 四层音频（BGM/SFX/UI/环境）框架
└── SaveSystem   user:// JSON 存档

继承体系
├── BaseWeapon ← magic_bolt / fire_ring / lightning_chain
└── BaseEnemy  ← slime_small / slime_big / bat / skeleton / ghost
                 （boss_shadow_lord 独立实现 CharacterBody2D）

场景树（main.tscn）
Main(Node2D, 主控/背景/震屏/BOSS计时)
├── Player（含 Camera2D[group:camera]、PickupArea）
├── EnemySpawner
├── HUD / LevelUpPanel / GameOverPanel / PauseMenu / DamageNumbers（均 CanvasLayer）
```

**架构优点**（保留并强化）：事件驱动解耦、数据驱动配置（难度/被动/词条为常量字典）、UI 全代码构建规避 `.tscn` 节点缺失、存档字段级合并向后兼容。

**架构待办**：见 §7「技术规范」——重点是 §7.3「单一数据源原则」与 §7.4「对象池接入」。

## 0.3 ⚠️ 设计 ↔ 实现脱节修正清单（统一标准）

> 以下为本次重做**拍板的统一标准**。各系统章节按此口径书写；代码中不一致处列入路线图 P0/P1。

| # | 脱节点 | 现状 | **v3 统一标准** |
|---|--------|------|----------------|
| D-1 | **玩家数值数据源** | Player 节点与 GameState 各存一份 HP/移速，仅开局同步，局内被动只改 GameState | **GameState 为唯一权威**。Player 每物理帧从 GameState 读 `move_speed`/`armor` 等；HP 以 `GameState.current_hp/max_hp` 为准，Player 不再自存。详见 §7.3 |
| D-2 | **暂停机制** | ESC 暂停只设 `is_paused` 标志，玩家/敌人物理不检查 → 不真正暂停 | 统一用 `get_tree().paused = true` + 节点 `process_mode`；升级/结算/暂停面板设 `ALWAYS`，游戏对象设 `PAUSABLE`。`GameState.is_paused` 降级为只读镜像 |
| D-3 | **超武合成规则** | magic_bolt 正确（Lv7+被动MAX→Lv8）；fire/lightning 在 Lv7 即自动套超武且 `max_level=7` 上不去 | **全武器统一**：武器 Lv7(MAX) + 指定被动 Lv5(MAX) → 合成，武器进入 **Lv8 超武形态**。所有武器 `max_level=8`，超武数值只在 Lv8 生效 |
| D-4 | **词条作用域** | 暴击/吸血词条被加到全局 `GameState`；贪婪(exp)词条未处理 | 区分**武器级词条**（伤害/攻速/该武器暴击/该武器吸血，存于武器实例）与**全局词条**（轻盈移速/贪婪经验/坚毅减伤，存于 GameState）。详见 §3.2.3 |
| D-5 | **暴击结算** | 投射物只读 `GameState.crit_chance`，忽略武器自身暴击 | 攻击结算时 `暴击率 = 全局基础(锐眼) + 武器级暴击词条`，由武器把最终暴击率传给投射物 |
| D-6 | **BOSS 数值** | 代码 HP=800/经验=200；原 GDD HP=500/经验=500 | 采用 **HP=800**（10 分钟登场时玩家通常已成型）、P2=50%、P3=25%、**经验=500**、击杀掉落复活羽毛+宝箱 |
| D-7 | **击杀 BOSS → 胜利** | 关卡模式击杀 BOSS 不触发胜利结算 | 关卡模式击杀 BOSS **立即触发胜利结算**，之后转入无尽模式继续可玩 |
| D-8 | **伤害随机浮动** | 烈焰环被乘了两次 ±10% | 浮动仅在 `get_damage_with_bonus()` 统一施加一次 |
| D-9 | **等级上限** | 50 级后经验仍增长 | ✅ 50 级后停止累计经验、不弹窗（`_on_exp_collected` 提前 return） |

> 在星屑坠落的荒原，操控法师在无尽魔物潮中杀出一条血路——**移动即战术，Build 即宿命，活到 15 分钟即是传奇**。

## 1.2 目标玩家

| 维度 | 描述 |
|------|------|
| 年龄 | 12–35 岁 |
| 经验 | 轻–中度，偏好短时快节奏 |
| 核心动机 | 爽感释放、Build 探索、分数攀比 |
| 单局时长 | 5–20 分钟（主力 15 分钟） |
| 平台 | PC 为主，移动端碎片化 |
| 分级 | ESRB E10+ / PEGI 7（卡通像素，无血腥） |

## 1.3 差异化定位

| 维度 | Vampire Survivors | Brotato | **星屑割草** |
|------|------|------|------|
| 攻击 | 全自动 | 手动+自动 | **全自动，武器进化改变形态** |
| 成长 | 武器+被动→超武 | 武器叠加 | **进化树 + 超武合成 + 随机词条** |
| 角色 | 初始武器不同 | 角色特性 | **专属被动 + 解锁剧情** |
| 地图 | 固定多图 | 小竞技场 | **种子随机 + 地形交互** |
| 局外 | 金币强化 | 角色解锁 | **完整 Meta + 图鉴收集** |
| BOSS | 定时刷 | 波次 | **多阶段 + 专属弹幕 + 台词** |

## 1.4 核心卖点（四大支柱，均为本次必保留项）

1. **武器进化树 + 超武合成** — 6 武器各有升级分支；武器 MAX + 指定被动 MAX → 终极超武（核心爽点）。
2. **Meta 局外永久成长** — 金币商店、角色/成就/图鉴解锁，让每局都有意义。
3. **多角色 + 解锁** — 法师 / 剑圣 / 弓手，各有专属被动与解锁条件、剧情台词。
4. **随机种子 + 地图机制** — 种子复刻分享、障碍物/宝箱、多地图地形交互。

辅以统一的「星尘魔法」美术语言（发光粒子）与多阶段 BOSS。

## 3.1 玩家系统 🟡

### 3.1.1 状态机

```
        ┌──────────┐
游戏开始→│  正常态  │← 无敌结束 / 升级结束 / 控制结束
        └────┬─────┘
   ┌────┬────┼────┬────┬────┐
   ↓    ↓    ↓    ↓    ↓    ↓
 受伤 升级暂停 死亡 减速 冰冻 眩晕
```

代码枚举 `PlayerState { NORMAL, HURT, LEVELING, DEAD, STUNNED }` ✅。减速/冰冻/中毒等异常状态以独立计时变量实现 🟡。

### 3.1.2 异常状态规则

| 状态 | 来源 | 效果 | 时长 | 叠加 | 状态 |
|------|------|------|------|------|------|
| 受伤无敌 | 被敌人接触 | 无视碰撞伤害 | 0.5s | 刷新计时 | ✅ |
| 升级暂停 | 升级触发 | 时间暂停 | 直到选择 | — | ✅ |
| 死亡 | HP≤0 | 游戏结束（可复活一次） | — | — | ✅ |
| 减速 | 冰系 | 移速 −30% | 2s | 至 −60% | 🟡 |
| 灼烧 | 火系（烈焰环 Lv7+）| 每秒固定 DOT（紫字）| 3s | 取最大 | ✅ |
| 冰冻 | 冰晶满级 | 完全冻结 | 1.5s | 不可叠 | ❌ |

### 3.1.3 回血 / 护盾 / 暴击 / 伤害类型

**回血来源** ✅🟡：回血球（+20% 最大 HP）✅ / 吸血词条（伤害转 HP）🟡 / 每秒回血被动 ✅ / 击杀回血 ❌ / 受伤后回血 ❌。所有回血不超过最大 HP。

**护盾** ✅：临时额外血条，受击先扣护盾再扣 HP；上限 50% 最大 HP；破盾不触发无敌帧。

**暴击** 🟡：基础 0%；暴伤 1.5×；来源＝锐眼被动（每级 +5%）+ 武器暴击词条（见 D-5）；暴击率上限 60%、暴伤上限 3.0×；视觉为金色数字 +「!」+ 略放大 ✅。

**伤害类型**（决定伤害数字颜色，`damage_number_requested` 第 4 参）✅：

| 类型 | 颜色 | 特点 | code |
|------|------|------|------|
| 物理 | 白 | 受护甲减免 | 0 |
| 魔法 | 蓝 | 无视护甲 | 1 |
| DOT | 紫 | 无视护甲，不可暴击 | 2 |
| 真实 | 金 | 无视一切（超武） | 3 |

**无敌帧与受伤反馈** ✅：无敌 0.5s；精灵红闪 3 次；震屏 4px/0.15s（BOSS 8px/0.3s）；死亡 0.3s 慢动作（`Engine.time_scale=0.3`）。

### 3.1.4 ⚠️ 数据源规范（D-1，本系统核心修正）

当前玩家与 `GameState` 各存一份 HP/移速，导致局内被动失效（B-3/B-4）。**v3 标准**：

- `GameState` 持有全部运行时数值（`current_hp/max_hp/move_speed/armor/crit_chance/...`）为**唯一权威**。
- Player 节点**不再自存** `current_health/max_health/move_speed`，改为每物理帧（或属性 getter）读取 GameState。
- 被动/Buff 只写 GameState，立即对玩家生效，无需手动同步节点。
- 详细数据流见 §7.3。

### 3.2.2 六种武器进化树

> 状态：6 种武器全部实现 ✅（各含 Lv8 超武，集成测试通过）。
> **超武统一规则（D-3）**：武器 Lv7(MAX) + 指定被动 Lv5(MAX) → 自动合成，武器进入 **Lv8 超武形态**。

#### ① 魔法弹 ✅（超武·星爆 / + 经验加成 MAX）
追踪光弹。Lv1 伤 12 / 攻速 0.8 → 逐级提伤害·攻速·穿透 → Lv7 穿透 2 + 分裂弹。**Lv8 星爆**：慢速大光球贯穿全屏，每 3s 全屏星屑爆炸（伤 40 / 穿透 5 / AOE）。

#### ② 烈焰环 ✅（超武·炼狱 / + 铁壁 MAX）
玩家为中心的燃烧环。Lv1 伤 8 / 范围 80 → 逐级提伤害·范围·频率 → Lv7 灼烧 DOT。**Lv8 炼狱**：三层火环 + 灼烧翻倍 + 范围内减速 20%。*(当前 `max_level=7` 且 Lv7 误触发超武，按 D-3 修)*

#### ③ 闪电链 ✅（超武·雷神之怒 / + 疾步 MAX）
连锁弹跳。Lv1 伤 8 / 连锁 2 → 逐级提伤害·连锁·弹跳范围 → Lv7 连锁 4 / 范围 220。**Lv8 雷神之怒**：连锁 6 + 弹跳召唤天雷（伤 30 / 范围 300）。*(同 ② 待修)*

#### ④ 冰晶风暴 ✅（超武·冰河纪元 / + 生命强化 MAX）
环绕冰晶。Lv1 伤 6 / 2 颗 / 30rpm → 逐级提伤害·数量·转速 → Lv7 命中减速 30%/2s。**Lv8 冰河纪元**：减速→冰冻 1.5s，冰冻碎裂造成范围伤害。

#### ⑤ 圣光之矛 ✅（超武·圣裁 / + 铁壁 MAX）
贯穿直线宽矛。Lv1 伤 20 / 宽 16 / 穿透 1 → 逐级提伤害·宽度·穿透 → Lv7 +15% 暴击。**Lv8 圣裁**：暴击 +30% + 暴伤 2.0× + 贯穿全屏。

#### ⑥ 毒雾 ✅（超武·瘟疫 / + 幸运 MAX）
地面 DOT 区域。Lv1 3/s / 60px / 3s → 逐级提伤害·范围·持续 → Lv7 中毒可叠 5 层。**Lv8 瘟疫**：毒雾跟随玩家 + 中毒层数无上限。

### 3.2.3 ⚠️ 词条体系（D-4 / D-5，统一作用域）🐞

升级时除固定成长外，独立 roll 稀有词条。**v3 区分两类作用域**：

| 词条 | 稀有度 | 概率 | 作用域 | 效果 |
|------|--------|------|--------|------|
| 锐利 | 普通 | 15% | 武器级 | 该武器伤害 +10%（叠 3） |
| 迅捷 | 普通 | 15% | 武器级 | 该武器攻速 +10%（叠 3） |
| 吸血 | 稀有 | 5% | **武器级** | 该武器伤害 3% 转 HP（叠 3） |
| 暴击 | 稀有 | 5% | **武器级** | 该武器暴击率 +8%（叠 3） |
| 轻盈 | 史诗 | 2% | 全局 | 移速 +5%（叠 2） |
| 贪婪 | 史诗 | 2% | 全局 | 经验 +10%（叠 2）*(当前无效，待修)* |
| 坚毅 | 史诗 | 2% | 全局 | 减伤 +2（叠 2） |

**结算口径**：攻击时 `最终暴击率 = GameState.crit_chance(锐眼) + 武器实例暴击词条`；命中后吸血按「武器级吸血 + 全局吸血」分别计入。投射物需携带武器最终暴击率（修 D-5）。

## 3.4 伤害计算公式 ✅🟡

```
最终伤害 = 基础伤害 × (1 + 增伤之和) × 暴击判定 − 护甲减免（按伤害类型）

  暴击判定：randf() < 最终暴击率 → ×暴伤倍率（DOT 不可暴击）
  护甲减免：物理 = max(1, dmg − max(0, 敌护甲 − 穿透))；魔法/真实无视护甲
  随机浮动：×randf(0.9, 1.1)（仅施加一次，D-8）
```

```gdscript
# DOT：不暴击、不受护甲
DOT总伤害 = 单层伤害 × 层数 × (持续时间 / 伤害间隔)   # 灼烧/中毒间隔 1.0s

# 近战 AOE（烈焰环/环绕剑刃）：距离衰减
AOE伤害 = 武器伤害 × max(0.5, 1 − 距离/最大范围)        # 边缘最低 50%
```

## 7.3 ⚠️ 单一数据源原则（D-1，最重要的技术约束）

**问题**：当前 Player 与 GameState 各存 HP/移速，被动只改 GameState，玩家不同步 → 幻血、移速被动无效。

**规范**：
1. 所有「会被被动/Buff/Meta 修改」的玩家数值，**只存在于 `GameState`**。
2. Player 通过属性读取，不缓存：
   ```gdscript
   # player.gd —— 不再有 var move_speed / var current_health
   func _handle_movement() -> void:
       var dir := Input.get_vector("move_left","move_right","move_up","move_down")
       velocity = dir * GameState.effective_move_speed()   # 含减速/Buff
   ```
3. 减速/Buff 作为「乘区」叠在 `GameState.effective_move_speed()` 内计算，不再写常量 `BASE_SPEED`。
4. HP 同理：`take_damage/heal` 直接读写 `GameState.current_hp/max_hp`，`_die()` 判 `GameState.current_hp<=0`。
5. HUD 已基本读 GameState，统一后两边自然一致。
